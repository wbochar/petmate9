Color Bars Feature Notes (pause/resume handoff)
Date: 2026-05-18

1) What drives Tools > Color Bars
- Menu entries are defined in `public/menu.js`.
- `colorBarsDocuments` is the ordered list of frame titles shown in the submenu.
- `buildColorBarsSubmenu()` sends:
  - `open-color-bars` with the selected frame title (single-frame open)
  - `open-all-color-bars` (load all bundled frames)

2) Runtime open flow
- Renderer handlers are in `src/index.ts`.
- `getBundledColorBarsWorkspace()` loads and caches:
  - `assets/colorbars_workspace.petmate`
- `openBundledColorBarsFrame(frameTitle)`:
  - prompts for unsaved changes
  - finds frame by exact `name` match
  - loads a one-frame workspace
- `openAllBundledColorBarsFrames()`:
  - prompts for unsaved changes
  - loads all frames from the bundled workspace

4) Important coupling rules
- Menu frame labels must exactly match the frame `name` in the workspace file.
- If you rename/add/remove a frame in `assets/colorbars_workspace.petmate`, also update `colorBarsDocuments` in `public/menu.js`.
- Keep frame order stable if index-based tooling/scripts are used during edits.

5) Generator script (optional, can overwrite manual edits)
